# Presentation Design Skill

> 本 skill 定義 Claude 在協助簡報製作時的思考方式、檢查流程與建議邏輯。
> 深度參考資料位於 `references/` 目錄，遇到特定主題時可查閱。

## ⚠️ Skill 串接規則

製作投影片時，必須遵守以下順序：
1. **先用本 skill（presentation）** 完成設計規劃（結構、文字量、版面、配色決策）
2. **再選擇執行工具產出投影片：**
   - **路徑 A：`anthropic-skills:pptx` skill** — 用 pptxgenjs 產出 .pptx 檔案
   - **路徑 B：Canva Pro（Chrome 自動化）** — 透過 Claude in Chrome 在 Canva 編輯器中操作

### 路徑 B：Canva Pro 工作流程
1. 在 Canva 中選擇或建立簡報專案
2. 依本 skill 的設計規劃，逐張操作：選模板 → 修改文字 → 調整配色與版面
3. 每張完成後截圖確認，對照本 skill 的 per-slide checklist（Section C）
4. Canva 編輯器 DOM 較複雜，若自動化遇阻，改為：產出詳細設計稿讓使用者手動排版

### 設計原則優先權
無論使用哪條路徑，當執行工具的內建設計建議與本 skill 的原則衝突時，**以本 skill 的原則為準**。

## A. 核心原則（15 條）

以下原則按優先順序排列。衝突時，上方原則優先。

### A1. 一張投影片 = 一個訊息
每張投影片只承載一個核心訊息。觀眾應能在 15 秒內讀完。

### A2. 標題 ≤ 10 字
中文標題控制在 10 個字以內（Miller's Law）。超過則拆成主標（主張）+ 副標（背景/依據），各 ≤ 10 字。

### A3. 單張字數 ≤ 120 字
以條列取代段落。關鍵字優先選擇：數字 > 專有名詞 > 衝擊性詞語。

### A4. 留白 40%
整張投影片留白約 40%（上下左右各約 10%）。寧可分頁也不壓縮字體。不足時用行距、字距、標題間距補足。

### A5. Z 字形排版
資訊順序遵循「左上 → 右上 → 左下 → 右下」的 Z 字形視線動線。
- 標題放最上方（字級最大，視線起點）
- 結論/重點放視線先到處，不要埋在底部
- 圖像放左側，文字放右側

### A6. 標題與內文字級比 1:1.5
推薦組合：36/24（小場地）、42/28（一般）、60/40（大場地）。
行距 = 字級的 50-75%，字距 = 字級的 25%。

**Canva 預設設定（使用者偏好）：**
- 字體：Noto Sans T Chinese
- 每頁標題 60 pt / 內文 40 pt
- 字母間距：25（Canva 單位，≈ 字級的 25%）
- 行距：1.5（Canva 倍數，≈ 字級的 50%）
- 配色：白底黑字，重點字使用紅色

### A7. 每張投影片 ≤ 3 色
重點色面積 ≤ 10%。顏色必須承載語意（紅=警示、綠=正面/環保、藍=信賴），不為裝飾而上色。非重點元素用灰階。

### A8. 視覺優先於文字
文字+影像的 72 小時記憶留存率 65%，純文字僅 10%（Dual Coding Theory）。
能用圖像表達的就不用文字；複雜因果/流程/比例必須視覺化。

### A9. 設計一致性（Law of Similarity）
全篇統一：版型、字型字級、配色數量、bullet 符號、icon 風格、動畫效果。
細節的不一致會立刻被觀眾察覺。

### A10. 簡報 ≤ 10 頁
理想 8 頁，上限 10 頁（每張 2 分鐘 ≈ 15-20 分鐘專注區間）。
候選主題過多時，用「成對比較法」篩選。

### A11. 讓觀眾覺得是「自己做決定」（Nudge）
不用命令式語句。用選項、數據、提問引導觀眾自主得出結論。
利益衝突點是觸發心理抗拒的最高風險處，優先套用 nudge 設計。

### A12. 損失框架 > 利益框架
「不做會失去什麼」比「做了會得到什麼」更有說服力（Loss Aversion）。
負面訊息只聚焦一個痛點，且必須以正面形象收尾。

### A13. 標示進度與終點
每張投影片顯示頁碼或進度條。封面後放目錄，章節切換時重新顯示目錄並 highlight 當前章節。

### A14. 主詞要明確
每個關鍵句子的主詞必須清楚。省略主詞僅限刻意營造客觀感的情境。
三種策略：「我」（個人意志）、「公司」（組織立場）、省略（客觀事實）。

### A15. 項目清單 3-5 項
所有項目須維持同一層次、相近字數、一致密度、統一詞性結尾、邏輯排序。
超過 6 項必須拆頁或濃縮。

## B. 工作流程

### Phase 1: 主題與結構（骨架）

1. **釐清目的**：這份簡報要觀眾「知道什麼」或「做什麼」？
2. **列出候選主題**：自由發散，不限數量
3. **篩選至 ≤ 10 項**：用成對比較法（→ ref `09`）
4. **決定敘事弧線**：
   - 開場：負面事實/挑釁問題 建立問題意識（→ ref `29`）
   - 中段：證據、選項、比較（→ ref `25`, `26`）
   - 收尾：解方 + 正面數據 + 行動呼籲（→ ref `30`）
5. **製作目錄頁**：揭露整體架構

### Phase 2: 內容設計（血肉）

6. **每張投影片寫出一句話核心訊息**（先不管版面）
7. **標題優化**：套用 10 字規則（→ ref `01`）
8. **內容精簡**：≤ 120 字，3-5 個 bullet（→ ref `11`, `06`）
9. **判斷圖像化**：
   - 數字/比例 → 圖表
   - 流程/順序 → 流程圖
   - 實體物品 → 照片/插畫
   - 學術專業概念 → 文字為主 + 局部圖像
   （→ ref `18`, `19`）
10. **說服設計**：
    - 提案頁：準備 3 方案（目標 + 高低誘餌）（→ ref `26`）
    - 關鍵 takeaway：考慮階段式揭露（→ ref `27`）

### Phase 3: 視覺設計（外觀）

11. **版面配置**：Z 字形排版（→ ref `04`, `12`）
12. **留白檢查**：40% 原則（→ ref `02`, `10`）
13. **視覺重心**：四等分法檢查平衡（→ ref `13`）
14. **配色**：≤ 3 色，語意配色（→ ref `14`, `15`）
15. **字型**：無襯線體為主，襯線體僅用於引用/強調（→ ref `05`）
16. **圖表設計**：預設灰階 + 重點上色，移除多餘格線（→ ref `20`, `21`）
17. **表格設計**：減法框線，用留白/網底取代（→ ref `22`）
18. **圖片配置**：對齊邊緣、等距、人物視線朝向文字（→ ref `24`）

### Phase 4: 檢查與修正

19. **逐頁檢查清單**（見 Section C）
20. **整體一致性檢查**（→ ref `17`）
21. **進度標示確認**（→ ref `16`）
22. **反白/強調使用確認**：封面/標語/關鍵數字才用（→ ref `08`）
23. **縮圖檢視**：把所有頁面縮小排列，檢查整體節奏與重心

## C. 逐頁檢查清單

每張投影片完成後，過一遍：

### 文字
- [ ] 標題 ≤ 10 字？
- [ ] 總字數 ≤ 120 字？
- [ ] 主詞明確？
- [ ] 項目清單 3-5 項、層次一致、詞性統一？

### 版面
- [ ] 留白 ≈ 40%？
- [ ] 資訊流向符合 Z 字形？
- [ ] 視覺重心平衡（四象限檢查）？
- [ ] 圖像在左、文字在右？

### 色彩
- [ ] ≤ 3 色？
- [ ] 重點色 ≤ 10% 面積？
- [ ] 顏色語意正確（紅=警示、綠=正面…）？
- [ ] 文字與背景明度差足夠？

### 圖表（如適用）
- [ ] 圖表類型正確（折線=趨勢、長條=排名、圓餅=比例）？
- [ ] 預設灰階 + 僅重點上色？
- [ ] 直接在圖上標註訊息，不依賴圖例？
- [ ] 表格框線最小化？

### 說服（如適用）
- [ ] 是否避免命令式語句？
- [ ] 是否提供選項而非單一結論？
- [ ] 負面訊息後是否銜接解方？
- [ ] 不適設計是否限制在 1-2 處？

## D. 反模式（常見錯誤）

| 反模式 | 為什麼錯 | 正確做法 |
|--------|---------|---------|
| 全頁塞滿文字 | 視覺壓力、認知負荷過高 | 精簡至 ≤ 120 字 + 圖像 |
| 每個元素都上不同顏色 | 等於沒有重點 | 灰階為底，僅重點上色 |
| 標題 15+ 字 | 超出短期記憶容量 | 拆主標 + 副標，各 ≤ 10 字 |
| 圖放右邊、文放左邊 | 違反 Z 字形視線 | 圖左文右 |
| 表格每格都畫框線 | 條紋壓力、視覺疲勞 | 僅標題列框線或斑馬紋網底 |
| 圓餅圖用 5+ 色 | 混亂無焦點 | 灰階底 + 強調項彩色 |
| 直接宣告結論 | 觸發心理抗拒 | 用數據/比較讓結論自然浮現 |
| 通篇恐懼訴求 | 觀眾疲乏、反感 | 負面開場 → 正面收尾 |
| 沒有頁碼或進度 | 觀眾焦慮「還有多久」 | 每頁標示進度 |
| 項目清單 7+ 項 | 超出認知負荷 | 拆頁或濃縮至 ≤ 5 項 |

## E. 決策指引

### E1. 文字 vs 圖像

```
內容能否轉為「任何人都能辨識的數據或形狀」？
├─ 是：數字/比例 → 圖表
│       流程/因果 → 流程圖
│       實體物品  → 照片/插畫
└─ 否：學術/技術專業 → 文字為主 + 具體範例局部圖像化
        抽象概念     → 圖文並用
```

### E2. 圖表類型選擇

```
想表達什麼？
├─ 時間趨勢變化      → 折線圖（橫軸連續）
├─ 項目排名/比較     → 長條圖（橫軸非連續）
├─ 各項目占比        → 圓餅圖（≤ 5 項，否則改長條圖）
├─ 流程/步驟/時序    → 時間序列圖
├─ 人物/組織關係     → 相關圖（主角置中）
└─ 多項目特性定位    → 矩陣圖（項目 ≥ 5 才適用）
```

### E3. 需要多色時怎麼辦

```
超過 3 色？
├─ 同類元素（如流程步驟） → 同色系漸層（深→淺或淺→深）
├─ 圖表多項目             → 灰階 + 僅重點彩色
│                           同色用實線/虛線或實心/網點區分
└─ 無法再減               → 前後頁用留白頁平衡視覺壓力
```

### E4. 分隔資訊的工具優先序

1. 留白（最優先）
2. 對齊
3. 背景色 / 網底
4. 文字粗細或顏色對比
5. 框線（最後手段，僅用於唯一重點）

### E5. 何時用反白文字（深色背景 + 白字）

- 封面、標語、關鍵數字、章節標題
- 推薦底色：黑 > 寶藍 > 紅 > 深綠
- 避免：黃底白字（對比不足）
- 放在投影片上半部（符合 Z 字形）
- 全篇不可過度使用，僅用於需優先吸睛處

### E6. 說服結構選擇

```
目的是什麼？
├─ 提案/推銷   → 3 方案比較（目標 + 高低誘餌）
│                 + 損失框架（不做會失去什麼）
│                 + 正面收尾
├─ 教學/知識   → 階段式揭露（先圖後字，延遲關鍵字）
│                 + 進度標示
├─ 警示/改革   → 恐懼開場（負面數據 + 挑釁問句）
│                 + 具體數字
│                 + 必須銜接解方
└─ 所有場景    → 避免命令式，用提問讓觀眾自答
```

## G. 臨床/學術簡報補充指引

本 skill 的主要使用者場景為臨床院內學術簡報、conference talk、教學投影片。
以下為針對此場景的額外考量：

1. **Case conference / teaching round（30+ 分鐘）**：
   章節切換目錄特別重要，每個段落重新顯示 agenda 並 highlight 當前位置。

2. **Guideline / CDSS 介紹**：
   進度條比頁碼更能降低「還有多久」的焦慮。

3. **住院醫師教學**：
   開場目錄 = 建立學習預期（成人學習原則）。
   學術專業概念不要強行圖像化，用具體臨床案例 + 局部圖解即可。

4. **恐懼訴求在醫療場景**：
   可用於強調疾病嚴重度或不治療的後果，但須注意倫理界線，避免過度煽動。
   負面數據後必須銜接治療/處置建議。

5. **學術圖表**：
   若需引用文獻圖表，必須標註出處。
   自製圖表遵循灰階 + 重點上色原則。
