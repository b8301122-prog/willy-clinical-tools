# #04 簡報請用 Z 字形來排版

> 來源：《你的簡報就是比 AI 強》
> 主題分類：對齊圖示 / 視線動向 / 版面配置

---

## 核心原則

人類的眼睛在閱讀橫式資料時,會依循「由左至右 → 移至左下 → 再由左至右」的 **Z 字形** 路徑移動。投影片排版應順著這個視線動向設計,觀眾才能更順暢地吸收資訊。

> **一句話記住**:讓版面上的資訊順序,符合眼球自然的 Z 字形走向。

---

## 為什麼要用 Z 字形排版

- **符合生理特性**:眼球運動(eye movement)天生傾向橫向掃描後下移
- **降低認知負擔**:視線不需要跳躍,閱讀更順暢
- **避免違和感**:若違反 Z 字形,觀眾會產生「怪怪的」(=違和感)
- **字級最大的標題**會最先映入眼簾,之後視線才會移至物件和下方的文字

---

## 正確 vs 錯誤的排版(BEFORE / AFTER)

### 範例一:純文字卡片排版

**❌ BEFORE(錯誤)**
```
[標題]
[方向性說明]
─────────────
[期待效果①]  [期待效果②]
─────────────
[問題①]      [問題②]
```
- 讀完標題後,視線被迫跳到**最下方**的要素(問題),
  再往上看期待效果 → 視線動向跳脫 Z 字形
- 結論放太下面,觀眾會先讀到細節才讀到重點

**✅ AFTER(正確)**
```
[標題]
[方向性說明]
─────────────
[背景①]      [背景②]      ← 先看背景(前提)
─────────────
[期待效果①]  [期待效果②]  ← 再看結論(效果)
```
- 把 BEFORE 的項目**上下對調**
- 視線從投影片上方依序往下誘導,形成自然的 Z 字形

---

### 範例二:含圖像(圖表、表格、照片)的排版

**❌ BEFORE(錯誤)**
- 圖像配置在**右側**,文字在左側
- 因為圖像會形成較大的視覺重點,視線會先被圖像吸引
- 導致無法做出 Z 字形的視線誘導

**✅ AFTER(正確)**
- 圖像配置在**左側**,文字配置在**右側**
- 先讓視覺重點(圖像)成為進入點
- 再讓觀眾透過右側文字進一步確認內容
- 更容易吸引觀眾的注意力

> **例外**:若希望放大展示照片或插畫,也可以將其配置在投影片**正中央**。但要注意可能發生排版協調的問題,需視具體情況調整。

---

## 排版檢查清單(可用於 Prompt / Skill)

建立或檢視投影片時,依序檢查:

- [ ] 標題是否在**最上方**(字級最大,作為視線起點)?
- [ ] 內容順序是否符合「上 → 下」「左 → 右」的 Z 字形?
- [ ] 結論/重點是否放在視線**先抵達**的位置,而非最下方?
- [ ] 若有圖像,是否放在**左側**(而非右側)?
- [ ] 若圖像放中央,排版是否仍保持協調?
- [ ] 讀者第一眼看到的元素,是否就是你希望他先注意到的?

## Prompt 使用範本

### 在 Claude 中請求投影片排版建議時

```
請依據「Z 字形排版原則」檢視以下投影片內容,並給出調整建議:

原則要點:
1. 視線動向遵循「左→右,上→下」的 Z 字形
2. 結論/效果應放在視線先抵達的上方,背景/前提放下方是錯誤的
3. 圖像應配置在左側,文字配置在右側
4. 字級最大的標題放最上方作為進入點

待檢視內容:
[貼上投影片元素與配置]
```

### 在 Claude Project Instructions 中加入

```markdown
## 投影片排版規則(Z 字形原則)

當協助設計投影片或簡報時,遵循以下排版原則:

- 內容排列順序必須符合 Z 字形視線動向(上→下、左→右)
- 標題放最上方,字級最大
- 「背景/前提」放上方,「結論/效果」放下方是錯誤的
  → 應將結論放在視線先抵達處
- 圖像(圖表、表格、照片)預設配置在**左側**,文字在右側
- 若圖像需放大展示可置於中央,但須確認排版協調
- 避免觀眾視線跳躍的配置
```

## 延伸要點

- 基本的 Z 字形有多種變化模式(單塊、多塊、含圖像等)
- 不論排版如何變化,核心都是**讓視線順暢地從左上走到右下**
- 商業簡報應盡量避免特殊排版,除非有明確目的
