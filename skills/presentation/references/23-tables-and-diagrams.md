# 投影片製作技巧：表格與圖解的運用

> **書名**：你的簡報就是比AI強:47個人類才懂的PPT技巧
> **章節**：4-6 無法用文字說明的概念就選用表格或圖解

---

## 核心原則

**無法用文字清楚說明的概念,就改用表格或圖解**。表格與圖解能表現機制、流程、關係等整體樣貌,大幅提高觀眾理解力。

---

## 一、何時該用表格 / 圖解取代文字

- 文字數量過多,導致簡報冗長難讀時
- 需要呈現「機制、流程、關係、結構」等整體樣貌時
- 需要呈現「階層、步驟、時間序列」等結構化資訊時
- 需要比較多個項目的特性時

---

## 二、表格的使用原則

### 用途

表格不只用於數據羅列或項目清單,**也可以用來簡潔地整理文字資訊**(目的、課題、對策、成果等條列式內容)。

### 製作訣竅(事前設定最重要)

1. **先釐清資訊的重要程度**:不要無頭緒地直接開表格
2. **依目的決定收錄多少資訊**:根據觀眾需要的資訊決定
3. **決定詳略程度**:只列重點,或需要連同詳細資訊一起羅列
4. **確認項目間的相互關係**:留意各欄位之間的對應邏輯

### 範例結構

| 欄位 | 內容 |
|---|---|
| 目的 | 提升自家公司產品的銷售額 |
| 小組構成 | 業務部的田中、鈴木、佐藤共三名 |
| 課題 | 經驗豐富的業務員較少 |
| 對策 | 舉行講座,由經驗豐富的成員來主講銷售 |
| 成果 | 全面提升銷售技能、營業額提升 30% |

---

## 三、圖解的基本原則

### 核心要素

- **符號**:利用箭頭(→)表現時間序列或流程
- **容器**:用方形或橢圓形文字框承載資訊
- **連接**:藉由連接方式表現彼此的關係性
- 必要時可從網路上下載免費素材符號使用

### 製作原則

1. **文字和顏色數量應限制在最小程度**,以追求更好的易讀性
2. **不需要花費太多時間製作**——圖解再漂亮,若無法傳達資料意圖就毫無意義
3. **務必要能傳達資料意圖**,圖解化能讓觀眾從「本能層次」直覺理解
4. 只利用箭頭符號與文字就能製成,實務上並不困難

---

## 四、三種常用圖解模式

### 模式 1:時間序列圖(流程圖)

**適用時機**:專案進度計畫、業務內容的流程、作業程序、部門階級結構

**製作步驟**:

1. 決定時間、日期、月分等時間軸的單位,依序排列於橫軸(或縱軸)
2. 把隨著時間變化的主題部分改成項目清單,再沿著時間軸逐一列出
3. 可把主題細分化,把多筆流程記載於同一張圖(例如同時呈現高麗菜、玉米、黃豆的作業流程)

**POINT**:

- 只要先畫出作為時間軸的橫軸,決定好流程數量,就能做出簡潔明瞭的圖解
- **注意:不要把不同的工作流程混在一起**

---

### 模式 2:相關圖

**適用時機**:組織結構圖、產業權力關係、部門關係、事件與事故因果關係

**適合表現**:

- ❶ 人物 & 企業的關係性(一眼理解某集團中每個人的關係)
- ❷ 行動的因果(當事件有多名參與者時,掌握事件發生的順序)

**製作步驟**:

1. 列出希望表現關係的相關者
2. 把關係性或利害關係的主要人物、行動因果的主要人物放在圖的正中央
3. 在相關者之間插入箭頭,標示出彼此的關係或行動

**POINT**:

- 不管相關者是人物或企業,都可以使用符號作為代表
- 標示出兩者之間的「社會性關係」或「職務上的影響」,再用箭頭或線條連接

---

### 模式 3:矩陣圖

**適用時機**:和其他公司比較商品特徵、說明自家公司產品或企業特性

**製作步驟**:

1. 設定縱軸和橫軸的評估軸
2. 定位對象的要素

**POINT**:

- 評估軸依縱軸和橫軸的不同觀點進行設定
- **定位的項目越多,矩陣圖解的資訊就會越多,效果自然就會越大**
- 反過來說,**如果項目只有 2–4 個,不使用矩陣圖會比較妥當**

---

## 五、決策 Checklist(可用於 prompt)

製作投影片時,依序自問:

- [ ] 這段內容能否用表格取代長段文字?
- [ ] 這段內容是否涉及時間順序或流程? → 用**時間序列圖**
- [ ] 這段內容是否涉及人物/組織間的關係或因果? → 用**相關圖**
- [ ] 這段內容是否涉及多項目的特性比較(且項目 ≥ 5)? → 用**矩陣圖**
- [ ] 圖解的文字和顏色是否已壓到最小?
- [ ] 是否只用箭頭與文字框就能完成,避免過度美化?
- [ ] 不同的工作流程是否混在同一張圖?(要避免)
- [ ] 表格的欄位是否事先釐清重要程度與相互關係?

---

## 六、可用於 Prompt / Skill 的精簡指令版

```
投影片內容設計規則:
1. 若文字過長或涉及機制、流程、關係,改用表格或圖解
2. 表格製作前先釐清:目的、重要程度、詳略、欄位關係
3. 圖解三大模式:
   - 時間序列圖:流程、步驟、進度計畫(橫軸為時間,不混流程)
   - 相關圖:人物/組織關係、因果(主要人物置中,箭頭連接)
   - 矩陣圖:多項目特性比較(項目 ≥ 5 才適用)
4. 圖解原則:文字與顏色最少化、用箭頭+文字框、不過度美化
5. 圖解的價值在於「傳達意圖」,不在於美觀
```
