# 簡報 Z 字形排版原則

> 書名：你的簡報就是比 AI 強：47 個人類才懂的 PPT 技巧
> 章節：2-4 簡報請用 Z 字形來排版

## 核心概念
人類閱讀橫式內容時，眼球會沿 **Z 字形** 移動：左上 → 右上 → 左下 → 右下。簡報排版應順應此視線動向，觀眾才能輕鬆閱讀。

## 基本規則

### 1. 標題永遠放上方
- 字級最大的標題先映入眼簾
- 觀眾看完標題後，視線才往下移動到內容區

### 2. 資訊由上而下依序排列
**❌ BEFORE（違反 Z 字形）**
- 標題下先放「方向性」與「期待效果」
- 再把「問題」放在最下方
- 結果：視線必須從標題跳到最下方再往上回讀，造成違和感

**✅ AFTER（符合 Z 字形）**
- 標題 → 方向性 → 背景 → 期待效果
- 由上而下一氣呵成，視線順暢

### 3. 圖像放左側、文字放右側
- 圖表、表格、照片是**視覺重點**，會先吸引目光
- 圖像在左 → 文字在右，形成 Z 字形視線誘導
- 若圖像放右側，會破壞 Z 字形動線（視線會卡在圖像上）
- **例外**：需要放大展示的主視覺圖，可置於投影片正中央

## 常見 Z 字形排版模式
1. 多欄並列（2-3 欄等寬）
2. 左圖右文
3. 上標題 + 下橫向流程
4. 上下分層（每層一個 Z）
5. 圖解公式型（A + B = C）

## 檢查清單
- [ ] 標題是否在最上方、字級最大？
- [ ] 重要資訊是否由上而下排列？
- [ ] 視覺重點（圖/表/照片）是否在左側？
- [ ] 視線是否能自然沿 Z 字形流動，無需回跳？
- [ ] 是否避免把關鍵元素孤立在右上或左下死角？

## 反模式（Anti-patterns）
- 🚫 標題下方直接放大型圖像在右側，文字在左側
- 🚫 重要結論放在投影片下半部，但上半部已有密集內容
- 🚫 多個視覺重點分散，沒有明確的視線起點與終點
