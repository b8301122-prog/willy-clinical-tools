# 靈活運用文字和圖像的標準

> **書名**：你的簡報就是比 AI 強：47 個人類才懂的 PPT 技巧
> **章節**：4-2

---

## 核心原則

**判斷是否圖像化的標準**：內容能否被轉換成「任何人都能辨識的數據或形狀」（可視化）。

---

## 適合圖像化的情境

### 1. 數字變動或比例 → 用圖表
- 純文字描述百分比（如「A 占 30%、B 占 17%、C 占 13%…」）難以理解
- 改用圓餅圖、長條圖可一眼看出項目多寡
- **範例**：Z 世代社會課題意識調查 → 用圓餅圖呈現「關心 56.8%」比純文字清楚

### 2. 順序或關係性 → 用流程圖
- 工作流程、步驟、因果關係適合用流程圖
- **範例**：達成交易的步驟
  - 潛在客戶 → 媒體傳播 → 公開研討會 → 電子報/DM → 諮詢/商談
- 純文字描述多步驟流程會變成冗長文章，觀眾難以閱讀

### 3. 商品示意圖、操作手冊 → 用照片或插畫
- 圖像說明比文字傳達更容易讓觀眾理解

---

## 不適合圖像化的情境

### 學術性專業用語、技術性資訊
- **原因**：這類資訊通常含有複雜且詳細的內容，需要專業知識或背景才能理解，難以簡化成一目了然的圖像
- **不要**強行把所有內容都轉成圖像

### 處理方式：圖像 + 文字並用
- 與其直接圖像化，不如用文字說明相關的**具體實例或應用範例**，再將一部分進行圖像化
- **範例**：說明「本能層次」（visceral）這個認知科學概念
  - ❌ 純文字：需長篇大論解釋 Donald Norman 的《情感@設計》三層次理論
  - ❌ 純圖像：抽象概念難以視覺化
  - ✅ 圖像 + 文字：用「非洲斑馬看到獅子會馬上逃跑」的具體例子 + 簡短文字說明

---

## 決策流程

```
內容類型判斷：
├─ 數字/比例變動？         → 圖表（圓餅圖、長條圖等）
├─ 順序/流程/關係性？      → 流程圖
├─ 實體物品/操作步驟？     → 照片或插畫
├─ 學術/技術性專業內容？   → 文字為主 + 舉具體例子輔以局部圖像
└─ 其他抽象概念？          → 圖像 + 文字組合說明
```

## 可直接複製到 Prompt 的精簡版

```markdown
## 投影片圖像化原則

**圖像化判斷標準**：內容能否轉成任何人都能辨識的數據或形狀。

**應該圖像化**：
- 數字/比例 → 圖表
- 流程/順序/關係 → 流程圖
- 實體物/操作 → 照片或插畫

**不應強行圖像化**：
- 學術專業用語、技術性資訊（含複雜細節，難以簡化）
- 抽象概念 → 改用「圖像 + 文字」：以具體實例輔助說明，再局部圖像化

**原則**：不是所有內容都要變成圖像。該用文字時用文字，該用圖像時用圖像，必要時兩者併用。
```
